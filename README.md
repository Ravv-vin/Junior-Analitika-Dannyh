## Блок 2. Python

### Задание 1. Изоморфизмы

```python
def is_isomorphic(s: str, t: str) -> bool:
    if len(s) != len(t):
        return False

    mapping_s_to_t = {}
    mapping_t_to_s = {}

    for char_s, char_t in zip(s, t):
        if char_s in mapping_s_to_t and mapping_s_to_t[char_s] != char_t:
            return False

        if char_t in mapping_t_to_s and mapping_t_to_s[char_t] != char_s:
            return False

        mapping_s_to_t[char_s] = char_t
        mapping_t_to_s[char_t] = char_s

    return True
```

Пример:
```python
s = 'paper'
t = 'title'
print(is_isomorphic(s, t))
```

Вывод:
```text
True
```

**Сложность:**
- Время: `O(n)`, где `n` — длина строки. Мы один раз проходим по символам.
- Память: `O(k)`, где `k` — количество уникальных символов. В худшем случае `O(n)`.

### Задание 2. Натуральная последовательность

```python
def missing_number(nums: list[int]) -> int:
    n = len(nums) + 1
    expected_sum = n * (n + 1) // 2
    actual_sum = sum(nums)
    return expected_sum - actual_sum
```

Пример:
```python
nums = [1, 2, 3, 4, 5, 6, 8, 9, 10, 11]
print(missing_number(nums))
```

Вывод:
```text
7
```

**Сложность:**
- Время: `O(n)`, потому что нужно один раз посчитать сумму элементов.
- Память: `O(1)`, потому что дополнительные структуры данных не создаются.

### Задание 3. Факторизация

```python
def prime_factors(n: int) -> list[int]:
    factors = []
    divisor = 2

    while divisor * divisor <= n:
        while n % divisor == 0:
            factors.append(divisor)
            n //= divisor
        divisor += 1

    if n > 1:
        factors.append(n)

    return factors
```

Пример:
```python
n = 56
print(prime_factors(n))
```

Вывод:
```text
[2, 2, 2, 7]
```

**Сложность:**
- Время: `O(sqrt(n))` в худшем случае, потому что достаточно проверять делители до квадратного корня из текущего числа.
- Память: `O(m)`, где `m` — количество найденных простых множителей в ответе.

### Задание 3. Покупки

Запрос:
```sql
SELECT
    a.client_id
FROM account AS a
LEFT JOIN transaction AS t
    ON t.account_id = a.id
    AND t.transaction_date >= CURRENT_DATE - INTERVAL '1 month'
    AND t.type = 'BUY'
GROUP BY a.client_id
HAVING COALESCE(SUM(t.amount), 0) < 5000;
```

Пояснение:
- `LEFT JOIN` нужен, чтобы не потерять клиентов, у которых за последний месяц покупок не было.
- Условия по дате и типу операции стоят в `ON`, чтобы `LEFT JOIN` не превратился в обычный `INNER JOIN`.
- `COALESCE(SUM(t.amount), 0)` заменяет `NULL` на `0` для клиентов без покупок.

Если в конкретной базе данных слово `transaction` считается служебным, имя таблицы нужно экранировать, например в PostgreSQL:
```sql
SELECT
    a.client_id
FROM account AS a
LEFT JOIN "transaction" AS t
    ON t.account_id = a.id
    AND t.transaction_date >= CURRENT_DATE - INTERVAL '1 month'
    AND t.type = 'BUY'
GROUP BY a.client_id
HAVING COALESCE(SUM(t.amount), 0) < 5000;
```

### Задание 2. Ручной счёт ROC-AUC

Данные содержат:
- 7 объектов положительного класса;
- 8 объектов отрицательного класса;
- всего `7 * 8 = 56` пар «положительный объект — отрицательный объект».

ROC-AUC можно посчитать как долю пар, где положительный объект получил оценку вероятности выше, чем отрицательный.

Количество правильно упорядоченных пар:
```text
42
```

Тогда:
```text
ROC-AUC = 42 / 56 = 0.75
```
